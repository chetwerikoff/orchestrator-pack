      REVIEW RUN IDEMPOTENCY (Issue #98, widened Issue #189 — additive to EMPTY REVIEW TRAP).
      Before calling ao-review run for a worker session, read Get-AoReviewRuns and the
      current PR head sha (from gh pr view or the worker PR ref). A head SHA is covered —
      do not spawn a new run — when any run matches the SAME prNumber linkage AND the EXACT
      normalized head SHA (targetSha, case-insensitive) and is in-flight (queued, preparing,
      running, reviewing) OR covered terminal (up_to_date, changes_requested). A run on
      the same SHA for a different PR does NOT count as coverage. A run on the same PR with a
      different targetSha does NOT count as coverage for the current head. Only a head with no
      run at all, or whose only runs on that PR+head are outdated, may use the plain uncovered
      start path.

      FAILED / CANCELLED ON CURRENT HEAD (not covered — not plain uncovered). A failed or
      cancelled run on the current head is NOT a covered terminal status (never treat
      findingCount: 0 on failed as clean). Do not start a new run through the plain uncovered
      path; follow EMPTY REVIEW TRAP: read latestRun.body (failure detail), retry at most once after
      diagnosis, escalate otherwise.

      AUTOMATED REVIEW-START CLAIM (Issue #267, closes the #189/#207/#235 concurrent
      window). Every automated starter surface must acquire the shared machine-local

      review-trigger-reconcile.ps1, orchestrator-wake-listener.ps1, review-trigger-reeval.ps1,
